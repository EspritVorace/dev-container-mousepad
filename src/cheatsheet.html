RCT-01
⚛ Hooks React
useState— état local, provoque un re-rendu à chaque mise à jour
const [val, setVal] = useState(initial)
setVal(prev => prev + 1) — mise à jour fonctionnelle (recommandé)
useEffect— effets de bord : appels API, abonnements, timer, DOM
useEffect(() => { ... }, [deps]) — ré-exécuté quand deps changent
useEffect(() => { ... }, []) — une seule fois au montage
return () => nettoyage() — appelé au démontage ou avant ré-exécution
useRef— référence mutable, persiste entre rendus, pas de re-rendu
const ref = useRef<HTMLDivElement>(null) — accès DOM via ref.current
useMemo— met en cache le résultat d'un calcul coûteux
useMemo(() => calculLourd(a, b), [a, b])
useCallback— met en cache une fonction (évite re-création à chaque rendu)
useCallback((x) => traiter(x, id), [id])
useContext— lit la valeur du Provider parent le plus proche
const thème = useContext(ThèmeContext)
useReducer— état complexe avec actions typées (alternative à useState)
const [état, dispatch] = useReducer(reducer, étatInit)
dispatch({ type: 'AJOUTER', payload: item })
useId— identifiant unique stable côté serveur et client (SSR-safe)
useLayoutEffect— comme useEffect mais synchrone avant le paint (mesures DOM)
useTransition— marque une mise à jour comme non-urgente (React 18+)
const [enAttente, startTransition] = useTransition()
useDeferredValue— retarde une valeur pour prioriser l'UI (React 18+)
TS-01
◇ TypeScript
Types primitifs
string · number · boolean · null · undefined · symbol · bigint
Array<T> ou T[] — tableau typé
[string, number] — tuple : taille et types fixes
Record<string, T> — objet dictionnaire clé → valeur
unknown — type sûr, forcer vérification avant usage
never — type impossible : fonctions qui throw, branches exhaustives
Interface vs Type
interface User { nom: string; âge?: number } — extensible
type ID = string | number — union de types
type Complet = Base & Extra — intersection : combine les deux
Interface → héritage, déclaration mergeable / Type → unions, tuples, mapped
Génériques
function identité<T>(arg: T): T — type paramétrique
<T extends { id: string }> — contrainte : T doit avoir id
<T, K extends keyof T> — K est une clé de T
Types utilitaires
Partial<T> — tout optionnel · Required<T> — tout obligatoire
Readonly<T> — propriétés en lecture seule
Pick<T, 'a'|'b'> — garde seulement a et b
Omit<T, 'secret'> — enlève la propriété secret
ReturnType<typeof fn> — type de retour d'une fonction
Parameters<typeof fn> — tuple des types de paramètres
Exclude<U, T> · Extract<U, T> · NonNullable<T>
Rétrécissement (narrowing)
typeof x === 'string' — garde primitif
x instanceof Classe — garde de classe
'prop' in obj — vérifie la présence d'une propriété
function isChat(a: Animal): a is Chat — garde personnalisée
as const — littéral immuable · satisfies T — valide sans élargir

⎇ Git & Terminal
Initialiser

git clone url — copie un dépôt distant
Cycle de base
git add -A — indexe tous les changements
git add -p — indexer interactivement par hunk
git commit -m "msg" — enregistre un instantané
git commit --amend — modifie le dernier commit
git push — envoie vers le distant
git pull --rebase — récupère + rebase (historique propre)
git fetch — récupère sans fusionner
Inspecter
git status — fichiers modifiés / indexés / non suivis
git log --oneline --graph — historique compact + branches
git diff — diff non indexé · --staged — indexé
git blame fichier — qui a modifié chaque ligne
Branches
git switch -c nouvelle — créer et basculer
git merge branche — fusionne dans la courante
git rebase main — rejoue les commits sur main
git branch -d branche — supprimer (si fusionnée)
Corriger & annuler
git stash — met de côté les changements en cours
git stash pop — restaure les changements mis de côté
git reset --soft HEAD~1 — annule commit, garde fichiers indexés
git reset --hard HEAD~1 — annule tout (⚠ destructif)
git revert HEAD — commit inverse (sûr pour historique partagé)
git cherry-pick abc123 — applique un commit spécifique
git reflog — historique de HEAD (filet de sécurité)
Terminal utile

xargs · sed · awk · wc -l · du -sh · curl · jq

⇶ Java Stream API
Créer un stream
liste.stream() — à partir d'une collection
Stream.of("a", "b") — à partir de valeurs
Arrays.stream(tab) — à partir d'un tableau
Stream.generate(() -> Math.random()) — infini (limiter !)
Stream.iterate(0, n -> n + 2) — séquence infinie
IntStream.range(0, 10) — 0→9 primitif, pas de boxing
IntStream.rangeClosed(1, 10) — 1→10 inclus
"texte".chars() — IntStream des caractères
Files.lines(Path.of("f.txt")) — lignes d'un fichier (lazy)
Opérations intermédiaires (lazy, chaînables)
.filter(x -> x > 0) — garde les éléments qui passent le test
.map(x -> x.getNom()) — transforme chaque élément
.mapToInt(User::getAge) — vers IntStream (évite boxing)
.flatMap(x -> x.getTags().stream()) — aplatit les sous-listes
.sorted() — tri naturel · .sorted(comparing(fn)) — tri custom
.distinct() — déduplique (equals) · .peek() — debug sans modifier
.limit(n) — n premiers · .skip(n) — ignore n premiers
.takeWhile(p) — prend tant que vrai (9+) · .dropWhile(p)
Opérations terminales (déclenchent le traitement)
.collect(Collectors.toList()) — List mutable
.toList() — List immuable (Java 16+, recommandé)
.collect(Collectors.toSet()) — Set
.forEach(e -> traiter(e)) — action sur chaque élément
.reduce(0, Integer::sum) — agrège avec valeur initiale
.reduce(Integer::sum) — retourne Optional
.count() · .findFirst() · .findAny()
.anyMatch(p) · .allMatch(p) · .noneMatch(p) — tests
.min(comp) · .max(comp) — extrêmes (Optional)
.toArray(String[]::new) — vers tableau typé
Collecteurs avancés
groupingBy(User::getVille) — Map<Ville, List<User>>
groupingBy(fn, counting()) — comptage par groupe
partitioningBy(u -> u.age() > 18) — Map<Boolean, List>
joining(", ") — concatène avec séparateur
toMap(User::id, User::nom) — Map (⚠ clés uniques)
toMap(k, v, (a,b) -> a) — gestion collision
summarizingInt(fn) — min, max, avg, sum, count
Parallèle
.parallelStream() — multi-thread (utile si > 10k éléments)
⚠ Éviter sur petites collections, E/S, ou opérations non thread-safe
